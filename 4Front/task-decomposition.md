### Задача 1.3: Настройка API клиента (Middle)
**Время**: 6-8 часов  
**Приоритет**: Высокий

**Описание**: Создать централизованный API клиент для взаимодействия с backend.

**Технические требования**:
- Создать Axios instance с базовой конфигурацией
- Реализовать interceptors для обработки ошибок
- Создать типы для всех API ответов
- Реализовать retry механизм

**Файлы для создания**:
- `src/services/api.ts` - Базовый API клиент
- `src/services/servicesApi.ts` - API для работы с сервисами
- `src/services/databasesApi.ts` - API для работы с БД
- `src/services/endpointsApi.ts` - API для работы с эндпоинтами
- `src/types/api.ts` - Типы для API

**Пример структуры API клиента**:
```typescript
// src/services/api.ts
import axios from 'axios';

const api = axios.create({
  baseURL: process.env.REACT_APP_API_URL || 'http://localhost:8080/api/v1',
  timeout: 10000,
});

// Interceptors для обработки ошибок
api.interceptors.response.use(
  (response) => response,
  (error) => {
    // Централизованная обработка ошибок
    return Promise.reject(error);
  }
);
```

**Критерии готовности**:
- [ ] API клиент корректно отправляет запросы
- [ ] Реализована обработка ошибок
- [ ] Созданы типы для всех API ответов
- [ ] Добавлены unit тесты для API функций

## Общие рекомендации для разработчиков

### Для Junior разработчиков:
1. **Начинайте с простых задач** - UI компоненты, формы, списки
2. **Изучайте TypeScript** - все компоненты должны быть типизированы
3. **Фокус на accessibility** - всегда думайте о клавиатурной навигации
4. **Тестируйте свой код** - пишите unit тесты для каждого компонента
5. **Следуйте стайл-гайду** - консистентность кода очень важна

### Для Middle разработчиков:
1. **Архитектурные решения** - думайте о масштабируемости
2. **Производительность** - оптимизируйте тяжелые операции
3. **Сложная логика** - API интеграция, состояние приложения
4. **Менторство** - помогайте Junior разработчикам
5. **Code review** - проверяйте код других разработчиков

### Общие принципы:
- **Mobile-first не нужен** - фокус на desktop опыт
- **Клавиатурная навигация** - приоритет номер один
- **Accessibility** - WCAG 2.1 AA соответствие
- **Performance** - быстрая загрузка и отзывчивый UI
- **Testing** - покрытие тестами критических сценариев

### Инструменты разработки:
- **VS Code** с расширениями для React/TypeScript
- **React Developer Tools** для отладки
- **Lighthouse** для проверки производительности
- **axe DevTools** для проверки accessibility
- **Storybook** для разработки компонентов
